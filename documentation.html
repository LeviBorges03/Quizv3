Introdução

Esta documentação descreve a estrutura e o funcionamento do Quiz de Biologia Celular. O objetivo é fornecer um guia claro sobre como o código está organizado e como cada parte contribui para a funcionalidade geral do quiz.

O quiz foi desenvolvido utilizando HTML para a estrutura, CSS para a estilização visual e JavaScript para a lógica interativa.

Estrutura HTML (index.html)

O arquivo index.html é a espinha dorsal da aplicação, definindo todos os elementos visíveis e a organização do conteúdo. Ele é dividido em várias seções principais:

Key attributes like id (for unique elements) and class (for grouping elements) are used extensively to allow specific styling via CSS and targeted manipulation via JavaScript.

  • Cabeçalho (Header): Contém o título do quiz.
  • Tela Inicial (Initial Screen): Apresenta o quiz ao usuário, com uma breve introdução e botões para escolher a dificuldade.
  • Interface do Quiz (Quiz Interface): Onde as perguntas são exibidas, junto com a barra de progresso e as opções de resposta. Esta seção fica oculta inicialmente.
  • Tela de Resultados (Results Screen): Mostra a pontuação final do usuário e um botão para reiniciar. Também oculta inicialmente.
  • Botão de Tema (Theme Switcher): Permite ao usuário alternar entre os temas claro e escuro.
  • Painel de Notificação (System Notification): Usado para exibir mensagens informativas, como dicas de ajuda.
  • Rodapé (Footer): Exibe informações de autoria.

Estilização CSS (estilo.css)

O arquivo estilo.css é responsável por toda a aparência visual do quiz. Ele define cores, fontes, layout, animações e a responsividade para diferentes tamanhos de tela. Alguns pontos chave incluem:

A classe .dark-theme é adicionada ao elemento <body> via JavaScript para aplicar as cores e estilos do tema escuro, que são definidos em conjunto com as variáveis de tema no CSS.

  • Reset Básico: Normaliza estilos padrão dos navegadores.
  • Temas Claro e Escuro: Classes CSS que alteram as cores de fundo, texto e elementos da interface. A alternância é controlada por JavaScript.
  • Layout Flexbox: Usado extensivamente para alinhar e distribuir elementos nas diferentes telas do quiz.
  • Estilos de Componentes: Definições para botões, barra de progresso, área de perguntas, notificações, etc.
  • Responsividade: Media queries para ajustar o layout e o tamanho dos elementos em telas menores, como as de celulares.
  • Animações e Transições: Efeitos sutis para feedback visual, como ao clicar em botões ou ao exibir respostas corretas/incorretas.

Exemplo de estilização para feedback visual em estilo.css:


/* Feedback visual para alternativa CORRETA em estilo.css */
.answer-option-button.correct-answer {
  border-color: #34c759; /* Borda verde */
  background: #eaffed; /* Fundo verde clarinho */
  color: #34c759; /* Texto verde */
  animation: pulse-correct 0.6s; /* Animacao de pulse! */
}

@keyframes pulse-correct { /* Animacao de "pulsar" pra resposta certa */
  0%   { box-shadow: 0 0 0 0 #34c75955; }
  70%  { box-shadow: 0 0 0 12px #34c75911; }
  100% { box-shadow: 0 0 0 0 #34c75900; }
}

Lógica JavaScript (index.js)

O arquivo index.js contém toda a interatividade e lógica do quiz. Suas principais responsabilidades são:

O fluxo principal do quiz é gerenciado por funções como startQuiz (para iniciar e configurar o quiz), displayQuestion (para mostrar a pergunta e alternativas atuais), handleAnswer (para processar a resposta do usuário, atualizar a pontuação e avançar para a próxima pergunta ou resultados), e showResults (para exibir a tela final). Funções auxiliares como updateProgressBar e showNotification cuidam de atualizar a interface do usuário dinamicamente.

  • Gerenciamento de Perguntas: Armazena a lista de perguntas, embaralha e seleciona conforme a dificuldade escolhida.
  • Controle de Estado do Quiz: Mantém o controle da pergunta atual, pontuação, e número total de perguntas.
  • Manipulação do DOM: Atualiza dinamicamente o conteúdo da página, como exibir perguntas, alternativas, mensagens de feedback e resultados.
  • Event Listeners: Responde a interações do usuário, como cliques em botões de dificuldade, alternativas de resposta, botão de ajuda e troca de tema.
  • Lógica de Resposta: Verifica se a resposta selecionada pelo usuário está correta, atualiza a pontuação e fornece feedback visual.
  • Barra de Progresso: Atualiza a barra de progresso e mensagens motivacionais.
  • Troca de Tema: Adiciona/remove classes CSS no corpo do documento para alternar entre os temas claro e escuro, e também gerencia o tema secreto. (Esta funcionalidade foi movida para documentation.js mas é chamada por index.js se a página principal também usar o tema).
  • Notificações: Exibe mensagens flutuantes para ajuda ou outras informações. (Esta funcionalidade foi movida para documentation.js).

Trecho da função que inicia o quiz:


function startQuiz(numberOfQuestions) {
    score = 0; // Zera pontuacao
    currentQuestionIndex = 0; // Volta pra primeira pergunta
    // ...
    selectedQuestions = shuffleArray([...quizQuestions]).slice(0, numberOfQuestions);
    totalQuestions = numberOfQuestions;

    initialScreenContainer.style.display = 'none';
    resultsScreenContainer.style.display = 'none';
    quizInterfaceContainer.style.display = 'flex';

    displayQuestion();
    updateProgressBar();
}

Exemplo de como a função displayQuestion atualiza o DOM:


// Em index.js
function displayQuestion() {
    const currentQuestion = selectedQuestions[currentQuestionIndex];
    questionTextElement.textContent = currentQuestion.question;

    answerOptionsContainer.innerHTML = ''; // Limpa alternativas anteriores

    currentQuestion.options.forEach((option, index) => {
        const button = document.createElement('button');
        button.className = 'answer-option-button';
        button.textContent = option;
        button.onclick = () => handleAnswer(index, button);
        answerOptionsContainer.appendChild(button);
    });
    updateProgressBar();
}

CSS da Documentação (documentation.css)

O arquivo documentation.css fornece os estilos específicos para esta página de documentação, garantindo uma apresentação clara e consistente com a identidade visual do quiz principal. Ele foi cuidadosamente elaborado para otimizar a legibilidade do conteúdo técnico.

Principais características do documentation.css:

  • Variáveis CSS para Temas: Define um conjunto de variáveis CSS para facilitar a aplicação de temas (claro e escuro), controlando cores de fundo, texto, links, e outros elementos da interface. Isso permite uma fácil customização e manutenção dos temas.
  • Estilização da Navegação (TOC): Formata a tabela de conteúdos para fácil navegação entre as seções da documentação.
  • Tipografia Otimizada: Ajusta fontes, tamanhos e espaçamentos para melhorar a leitura de parágrafos descritivos e títulos.
  • Formatação de Blocos de Código: Aplica estilos dedicados aos elementos <pre> e <code> para destacar snippets de código, utilizando cores de fundo e fontes monoespaçadas que facilitam a visualização.
  • Responsividade: Inclui media queries para adaptar o layout em diferentes tamanhos de tela, assegurando uma boa experiência tanto em desktops quanto em dispositivos móveis.
  • Consistência Visual: Reutiliza ou adapta elementos de design do estilo.css do quiz para manter uma identidade visual coesa, ao mesmo tempo que introduz estilos próprios para as necessidades de uma página de documentação.
